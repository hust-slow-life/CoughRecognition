> 最后更新：2026-01-21

## 一、产品定位

### 核心定位
- **医疗辅助决策工具**：帮助家长记录孩子咳嗽情况，结合 AI 提供初步诊疗建议

### 目标用户
- 有未成年孩子的家庭
- 使用者包括：父母、祖父母、保姆
- UI 需要尽可能精简，适合各年龄段使用者

### 目标孩子
- **年龄范围**：未成年人全覆盖（婴儿、幼儿、学龄前、学龄）
- **重要**：不同年龄段咳嗽声音特征差异大，需要针对特定年龄段优化模型

### 核心价值
- 快速记录孩子咳嗽声音
- AI 辅助分析咳嗽类型，帮助初步诊断
- 就医时给医生播放录音，提供客观数据
- 追踪病程发展

### 使用时机
- **孩子生病时才用**，不是长期监控工具
- 典型使用周期：3 天 - 2 周

### 平台
- iOS 应用（第一版）

## 二、核心使用场景

### 场景一：主动录制分析（核心场景）

**用户故事**：作为父母，当孩子正在咳嗽时，我希望能快速录下来，之后给医生听或让 AI 分析，帮助判断病情。

**典型流程**：
1. 孩子咳嗽 → 父母打开 App
2. 按大按钮开始录制（可以录第二次咳嗽）
3. 录制 10 秒内的咳嗽声
4. 保存，可选择立即分析或稍后分析

**需求点**：
- [x] 一键快速录制（大按钮，极简操作）
- [x] 录制时长：不到 10 秒的咳嗽声片段
- [x] 录制完成后可分析咳嗽类型
- [x] 分析结果供 AI 诊疗或医生参考

**咳嗽分类**：
- 基础分类：干咳、湿咳、连续咳、单声咳
- 扩展分类（尽可能细）：犬吠样咳（喉炎）、喘息性咳嗽、百日咳特征等
- 如识别到明确病症特征，可直接提醒

**设计原则**：
- 记录要非常非常简单——按一个大按钮迅速记下来
- 不需要第一时间分析，先记录下来最重要

---

### 场景二：后台监控模式

**用户故事**：作为父母，当我晚上睡觉或不在孩子身边时，希望手机能自动监控并记录孩子的咳嗽/哭泣，事后我可以查看。

**典型场景**：
- 晚上父母睡觉，听不到孩子咳嗽
- 父母在另一个房间
- 孩子单独在房间睡觉/玩耍
- 父母临时外出

**手机放置**：放在孩子旁边

**需求点**：
- [x] 手动开启/关闭监控（不需要自动开启）
- [x] 后台持续监听音频
- [x] 检测到咳嗽/哭泣时自动录制片段
- [x] **静默记录，事后查看**（不打扰父母睡眠）
- [x] 录音片段可逐个回放
- [x] 片段手动删除

**第一版暂不做**（放设置，后续迭代）：
- 连续多次咳嗽的特殊提醒
- 咳嗽/哭泣的优先级区分
- 推送通知功能

## 三、辅助功能

### 健康数据关联

**目的**：
- 追踪病程发展
- **核心需求：为 AI 辅助诊断提供更多信息**

**需求点**：
- 记录其他症状（发烧、流涕、咽痛等）
- 记录体温
- 记录用药情况
- 将症状与咳嗽记录关联

**设计原则**：
- 记录频率由父母自己决定
- 操作要简单快速

---

### 就医辅助

**使用场景**：
- 现场查看最近几天咳嗽发展状况
- 发送数据给 AI 做预诊判断
- 发送给医生做简单判断

**给医生展示**：
- **主要是播放咳嗽声音**（看诊时孩子可能没在咳嗽）
- 简单的病程报告

**给父母自己看**：
- AI 诊疗建议
- 可与医生建议对比，问诊时提供更多信息

**不需要**：
- 导出功能
- 复杂报告（只需简单报告）

---

## 四、非功能性需求

### 响应时间
- 不需要毫秒级响应
- 1 分钟内能识别即可
- 准确率优先于速度

### 数据隐私
- 第一版 Demo 暂不考虑
- 后续版本需要考虑本地处理 vs 云端处理

### 准确率目标
- 待定（需要根据数据集和模型测试确定基线）
- 咳嗽分类尽可能细，提高诊断参考价值

## 五、MVP 功能范围

### P0（必须有）
- [x] **一键快速录制**（核心交互，大按钮）
- [x] 咳嗽类型分类（干/湿/连续/单声，尽可能扩展更多类型）
- [x] 录音历史列表（可逐个播放）
- [x] 后台监控 - 咳嗽检测
- [x] 后台监控 - 哭泣检测
- [x] 自动录制片段（静默记录）
- [x] 孩子年龄设置（用于模型优化）

### P1（应该有）
- [ ] 健康日志记录
- [ ] 症状关联
- [ ] 简单病程报告
- [ ] AI 诊疗建议（接入外部 AI）

### P2（可以有）
- [ ] 趋势统计
- [ ] 就医提醒
- [ ] 连续咳嗽特殊提醒（设置项）
- [ ] 咳嗽/哭泣优先级设置

### 暂不考虑
- 云端同步
- 多设备支持
- 社交分享
- 医生端功能
- 数据导出
- 自动开启监控

## 六、开放问题

1. **咳嗽类型的医学定义**
   - 干咳、湿咳的区分标准是什么？
   - 是否需要咨询儿科医生确认分类标准？
   - 犬吠样咳、喘息性咳嗽等特殊类型的识别标准？

2. **儿童咳嗽的特殊性**
   - 不同年龄段（婴儿/幼儿/学龄前/学龄）咳嗽声音特征差异？
   - 现有数据集是否包含足够的各年龄段儿童样本？
   - 是否需要按年龄段训练不同模型？

3. **误报处理**
   - 如何处理环境噪音导致的误报？
   - 用户是否可以标记误报用于模型改进？

4. **AI 诊疗集成**
   - 如何接入外部 AI 进行诊疗建议？
   - 需要传递哪些数据给 AI？
   - 如何展示 AI 建议并做免责声明？

## 七、讨论记录

### 2026-01-21（第二次讨论）
- 完成 28 个需求澄清问题
- **产品定位升级**：从"记录工具"升级为"医疗辅助决策"
- **场景优先级调整**：主动录制成为核心场景

- **AI 角色明确**：帮助做初步诊断建议
- **咳嗽分类要求提高**：尽可能细，能识别特定病症最好
- **年龄适配重要**：不同年龄段需要针对性优化
- 详细澄清记录见 [需求澄清记录.md](需求澄清记录.md)

### 2026-01-21（第一次讨论）
- 初次讨论，确定基本方向
- 目标用户：小孩的父母
- 两个核心场景：后台监控 + 主动录制分析
- 需要关联健康数据
- 准确率优先，响应时间要求不高（1分钟内）
- 第一版不考虑数据隐私
- 功能和场景保持垂直，不复杂化
