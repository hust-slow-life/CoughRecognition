## 需求要点总结

### 产品定位
- **核心功能**：医疗辅助决策
- **目标用户**：有未成年孩子的家庭（父母、祖父母、保姆）
- **使用时机**：孩子生病时（3天-2周周期）

### 核心场景优先级
1. **主动录制**（核心）：一键快速录制咳嗽声，给医生或 AI 诊疗使用
2. **后台监控**：父母不在/睡觉时静默记录，事后查看

### 关键设计原则
1. **极简操作**：大按钮一键录制
2. **年龄适配**：不同年龄段咳嗽特征不同，需要针对性优化
3. **AI 辅助**：咳嗽分类尽可能细，帮助 AI 做初步诊断
4. **静默监控**：后台监控不打扰，只记录

### 技术要点
| 项目 | 决策 |
|------|------|
| 孩子档案 | 第一版单孩子，选择年龄段 |
| 后台监控 | 熄屏运行，前后各5秒，砍掉静默段 |
| 录音分析 | 云端分析，支持离线录制 |
| 多次咳嗽 | 分别识别每次咳嗽 |
| 录音分享 | 支持分享给医生 |
| AI 诊疗 | 第一版内置，需免责声明 |
| 历史记录 | 按天组织 |
| 简单报告 | 咳嗽次数 + 每片段频率 |

### 第一版可简化
- 多孩子档案 → 后续做
- 连续咳嗽特殊提醒 → 放设置，后续做
- 咳嗽/哭泣优先级区分 → 放设置，后续做
- 导出功能 → 不需要，只做简单报告
- 时间分布、类型分布 → 暂不需要
